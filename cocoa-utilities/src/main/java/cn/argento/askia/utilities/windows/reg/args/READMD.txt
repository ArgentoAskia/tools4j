build()移到Arg接口，因为所有的参数都是可选的，所以不需要参数的时候可以直接在Arg接口上进行build调用

所有的Arg子接口只负责两件事情：注册参数位（防止重复参数调用）、添加ArgInfo对象到AbstractRegCommand、返回OptionalArg子实现

所有的子命令类（如RegAdd）中，必须要自行指定好新的Arg内部类，如RegAddOptionalArg(public)，
其中这个类必须是OptionalArgs子类，并且实现必要的Arg子接口（适配参数），如ArgV<RegAddOptionalArg>、ArgVE<RegAddOptionalArg>
